    ⚠️ Billing Table Migration Required
    The Supabase billing table is missing columns needed for walk-in bills (is_adhoc, patient_name, patient_phone, etc.).
    Appointment-linked bills will still save (base columns). Walk-in bills cannot save until the migration is run.
    Fix: Open billing-migration.html in your browser and click "Run Step 2 — Add Columns", or run the SQL below in Supabase SQL Editor: 
    ALTER TABLE billing
  ALTER COLUMN appointment_id DROP NOT NULL,
  ADD COLUMN IF NOT EXISTS is_adhoc      BOOLEAN DEFAULT FALSE,
  ADD COLUMN IF NOT EXISTS patient_name  TEXT,
  ADD COLUMN IF NOT EXISTS patient_phone TEXT,
  ADD COLUMN IF NOT EXISTS patient_email TEXT,
  ADD COLUMN IF NOT EXISTS service       TEXT;
